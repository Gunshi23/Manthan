# Manthan: Autonomous Growth Marketing Command Center

Manthan is a real-time autonomous growth marketing command center. It integrates multi-agent AI coordination, database analytics telemetry, conversational voice uplink, and automated messaging campaign flows.

---

## 🛠️ Technology Stack

- **Frontend**: React 19 (TypeScript), Tailwind CSS 4, Framer Motion, Lucide React, Vite.
- **Backend**: Node.js, Express (TypeScript), Firebase Admin SDK (with a high-fidelity local `db_mock.json` fallback).
- **AI Integrations**: Gemini API (model `gemini-2.0-flash` on client and `gemini-2.5-flash` / `gemini-3.1-flash-lite` on server), Deepgram Voice Agent WebSocket API.
- **Delivery Services**: Twilio (WhatsApp/SMS dispatch), Resend (Email dispatch).

## 🚀 Key Subsystems & Implementations

### 1. Mission Control HQ Dashboard
Acts as the central operational headquarters of the growth network:
- **Telemetry Ledger**: Dynamic KPI cards displaying achieved revenue, forecasted revenue, growth rates, and conversion trends across customizable timeframes (Today, Week, Month).
- **AI Executive Briefing**: Embedded briefing panel displaying context-enriched summaries generated by the Gemini API from live campaigns, opportunities, and analytics.
- **Execution Pipeline Flowchart**: Visual real-time flowchart mapping active agent statuses during campaign creation (`Drishti (Audience)` → `Pragya (Opportunity)` → `Khoj (ROI)` → `Rachna (Copywriting)` → `Saarthi (Dispatch)`).
- **Pragya Opportunity Radar**: Displays hidden revenue leaks (cart drop-offs, inactive VIPs) with estimated potential gains, confidence indexes, and quick-launch campaign triggers.
- **Autonomous Demo Loop**: Loop utility that generates synthetic missions, triggers simulated agent courtroom debates, schedules conversions, and increments telemetry registers in real time.

### 2. Dynamic Credentials Synchronization
- Saving credentials in the header HUD configuration modal updates localStorage and POSTs credentials to `/api/config`.
- The backend writes the values to the workspace `.env` file and updates process variables in-memory, allowing updates to take effect immediately without backend server restarts.
- Frontend mount triggers automatically synchronize local client credentials to the backend.

### 3. Deepgram Voice uplink Drawer
- Adds a **VOICE NODE** indicator pill to the HUD status center.
- Clicking the pill opens a slide-out **Neural Voice Link** drawer containing sound wave animations, transaction state records, and call switches.
- Connects directly to `wss://agent.deepgram.com/agent?key={deepgramKey}` using the user-provided settings configuration payload.
- Captures browser microphone input at 48000Hz (PCM linear16), streams it to the WebSocket, and plays back returned audio chunks (PCM linear16 at 24000Hz) gaplessly via the Web Audio API (`AudioContext`).
- Extracts dialogue text messages (`type: "ConversationText"`) and renders them in a scrolling visual dialog log.

### 4. Firestore RESOURCE_EXHAUSTED Quota Fallback
- Encapsulates queries in the `/api/customers`, `/api/orders`, and `/api/campaigns` endpoints.
- If Firestore throws a `RESOURCE_EXHAUSTED` (gRPC code 8) quota limits exception, the server prints a warning, reads the static high-fidelity `db_mock.json` database backup, formats it into standard array responses to preserve the API schema structure, and returns HTTP 200.
- This keeps the entire system operational during live demonstration presentations even under heavy database load or exceeded quotas.
